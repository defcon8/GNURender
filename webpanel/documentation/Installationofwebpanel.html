Installation of webpanel

Getting Started ››

The binary repository for GNURender is hosted at GITHUB. The whole install system is

optimised to work with GIT for Linux. We suggest you use this tool to install GNURender,

and keep the directory structure the way we designed it. This makes it easy for you to

update in the future.


Install the necessary components:

       

       apt-get install git git-core

       

Get the latest version of GNURender:


       cd /opt

       git clone https://github.com/defcon8/GNURender

       

Now you'll need to create a virtual host entry in your Apache2 configuration. Edit

the file: /etc/apache2/sites-enabled/@000-default and add an entry as shown below:

       

       <VirtualHost *:80>

       ServerAdmin webmaster@mydomain.com

       ServerName mydomain.com


       DocumentRoot /opt/GNURender/webpanel

       <Directory />

               Options FollowSymLinks

               AllowOverride None

       </Directory>

       <Directory /opt/GNURender/webpanel>

               Options Indexes FollowSymLinks MultiViews

               AllowOverride None

               Order allow,deny

               allow from all

       </Directory>

       ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/

       <Directory "/usr/lib/cgi-bin">

               AllowOverride None

               Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch

               Order allow,deny

               Allow from all

       </Directory>

       ErrorLog ${APACHE_LOG_DIR}/error.log

       LogLevel warn

       CustomLog ${APACHE_LOG_DIR}/access.log combined

       </VirtualHost>


For more information on virtual hosts please read the manual of Apache2.


Because both the web- and database server needs to handle big (let's say: more than average) sized

files, you'll need to tune some parameters in your PHP and MySQL configuration files. While these settings

increase the usability of GNURender, they may be not so optimal for other websites. That's why we always

suggest you use a dedicated web server instance for GNURender.        


Edit (or if needed add) the following parameters


       php.ini (located in /etc/php5/apache2/)

       =======================================

       

       post_max_size = 1024M

       memory_limit = 1024M

       file_uploads = On

       upload_max_filesize = 1024M

       max_file_uploads = 20

       

       my.cnf (located in /etc/mysqld)

       ===============================

       

       [mysqld]

       max_allowed_packet = 1024M

       #bind-address = 127.0.0.1

       

       [mysqldump]

       max_allowed_packet = 1024M

       

After changing these parameters you'll need to reboot the packages:


       cd /etc/init.d

       apache2 restart

       mysqld restart

       

Now there is just one last thing you'll need to do. Under the webpanel's folder there

is a file called cron.php. This script is executing administrative tasks for the

GUI in the background. The script needs to be executed periodically. On Linux we

can use the crontable, Windows users can use the Windows Task Scheduler for this.

The task should be executed every 15 minutes. We find a interval of 15 minutes

to work just fine, you can decrease the interval to have more real-time statistics,

but this generates more load on your database server.


Edit the crontab by typing:


       crontab -e


And add a row:


       15      *       *       *       *       php /opt/GNURender/webpanel/cron.php


Make sure the path to the cron.php file is correct. After that press F2 (standard

pico/nano command for saving the file).


That's it! You should now point your web browser to:


       http://yourserver/install

       

This will launch the web based database configure program that will guide you through the

final steps of the installation. Do not forget to remove the install directory when

the installation is finished.
