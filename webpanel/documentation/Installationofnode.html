Installation of node

Getting Started ››

LINUX:


Install the necessary components:

       

       apt-get install mono-complete git git-core unzip


Then install ffmpeg:



       Get the latest static build from (http://ffmpeg.gusari.org/static/) and put the files in folder /usr/local/bin/

       Also make sure that the executable flags are set for these files. If needed do this manualy


               cd \usr\local\bin

               chmod +x ffmpeg

               chmod +x ffprobe


       Tip: Try running ffmpeg from the commandline without any parameters, Just to verify that the application is able to run propperly. The application should give some visible output to the console, like version number etc.



Step-1

------

If this node is on the same machine as the web panel, you already downloaded

GNURender and you can skip to step 2.


Get the latest version of GNURender:


       cd /opt

       git clone https://github.com/defcon8/GNURender        

       

Step-2

------          

Copy the file 'config.ini.dist' to 'config.ini' and modify it to your needs. The

file is self explaining (we think). It holds the database connection parameters and

optionally the path to the blender application executable. An empty line or a line

starting with # is a commented line and skipped while reading the configuration. The file

should look something like this:

         

               #        A unique node name.

               nodeid=My Test Node

               

               # Path and name of the Blender executable. The Path needs to end with a slash or backslash,

               # depending on the type of OS. If the path parameter is commented, the node assumes the

               # path to Blender is in the system environment PATH variable.

               #

               # (Windows example)

               # appdir=C:\Program Files (x86)\Blender\

               # appexec=Blender.exe

               #

               # (Linux example)

               # appdir=/opt/blender/

               # appexec=blender

               

               appdir=/opt/blender/

               appexec=blender


               #        MySQL database connection parameters

               mysqlhost=10.1.1.191

               mysqlport=3306

               mysqldb=test

               mysqluser=test

               mysqlpass=test

         

During the start-up of process node.exe, it will search for config.ini in it's start-up

directory. You can now start your node:        


       mono node.exe

       

The node is just an application that runs on a console. When you do this remotely through

a SSH connection, the process is terminated when you close the SSH connection.

A simple work-a-round is making use of an application named 'screen' which is a simple

wrapper that will continue running your process when the SSH session ends.

                       

That's it!
